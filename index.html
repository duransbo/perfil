Ideologias

Performance acima de tudo

Nenhum usuario permanece muito tempo em uma aplicação lenta, ninguem gosta de esperar. apesar de existir diversos outros fatores para que uma aplicação tenha uma boa usuabilidade, levar em conta a performance como o pilar principal para o desenvolvimento tras diversas vantagens para o desenvolvimento, pois sempre obriga o desenvolvedor a melhora-la.

Codigo de qualidade

Um gancho da ideologia acima, pois não é possivel atingir o maximo de performance com um codigo mal planejado. Codigo poluido, linhas desnecessarias, paradgma ineficiente e uma logica não funcional são caracteristicas de uma aplicação que normalmente possui uma performance ruim, dificil de dar manutenção e escalar.

Receio a frameworks

Frameworks são ferramentas que aceleram o desenvolvimento de aplicações para programadores preguiçosos(isso é uma qualidade), porém não são sempre a melhor solução. Geralmente diminuem a performance da aplicação por trazerem inumeras linhas de codigo que não serão utilizadas pelo desenvolvedor. É bem comum os resolver problemas de frameworks com gambiarras que poluem o codigo.
